## Метрика 1: Пожизненная ценность клиента (LTV) — упрощенная

### Определения из разных источников

| Источник | Определение | Откуда взято |
|----------|-------------|---------------|
| Финансы (1С) | Сумма всех платежей минус возвраты | 05_finance_payments.csv |
| Продажи (CRM) | Сумма всех закрытых сделок (Closed Won) | 02_crm_transactions.csv (связь с CRM) |
| Маркетинг | Прогнозная выручка (ARPU * Срок жизни) | Обсуждение команды |

### Количественный анализ

- Финансовое определение (Факт): ~2 500 000 рублей (на накопительную сумму по базе)
- Продажное определение (Потенциал): ~3 200 000 рублей
- Расхождение: ~20-25%
- Компании, где сделка закрыта, но денег нет: ~10-15 компаний

### Интерпретация

Продажное определение (CRM) систематически завышает LTV, так как считает потенциальную выручку по закрытым сделкам, которые могут быть еще не оплачены или отменены. Для оценки *реальной* эффективности каналов (задача Группы Б) важно считать только поступившие деньги.

### Предложение единого определения

| Поле | Значение |
|------|----------|
| **Метрика** | LTV (Упрощенный / Исторический) |
| **Определение** | Сумма всех успешных платежей (amount > 0) за вычетом возвратов (amount < 0), совершенных клиентами, пришедшими через данный канал, за весь период наблюдения |
| **Обоснование** | Для расчета ROI маркетинга необходимо использовать только реальные денежные потоки (Cash In), а не прогнозы или закрытые в CRM сделки без оплаты |
| **Алгоритм расчета** | SUM(amount) из файла 05_finance_payments, где company_id относится к данному каналу |
| **Владелец** | Финансовый департамент |

## Метрика 2: Конверсия в продажу (Conversion Rate)

### Определения из разных источников

| Источник | Определение | Откуда взято |
|----------|-------------|---------------|
| Маркетинг | Доля лидов со статусом 'Closed Won' | 07_marketing_leads.csv |
| Финансы (1С) | Доля лидов, по которым есть хотя бы один платеж | 07_marketing_leads + 05_finance_payments |
| Продукт | Доля лидов, которые совершили вход в LMS | 03_lms_events.csv |

### Количественный анализ

- Конверсия по Маркетингу (Closed Won): ~35%
- Конверсия по Финансам (Оплата): ~22%
- Конверсия по Продукту (Активность): ~18%
- Расхождение: 13% между Marketing и Finance

### Интерпретация

Существует значительный разрыв между "закрытой сделкой" и "фактической оплатой". Если использовать метрику маркетинга, эффективность каналов (например, контекстной рекламы) будет выглядеть завышенной, так как часть лидов конвертируется в сделку, но не доходит до оплаты. Для Группы Б приоритетом является привлечение *платящих* клиентов.

### Предложение единого определения

| Поле | Значение |
|------|----------|
| **Метрика** | Конверсия в продажу (CR) |
| **Определение** | Отношение количества уникальных компаний, совершивших хотя бы один платеж (amount > 0), к общему количеству лидов, привлеченных через данный канал |
| **Обоснование** | Отражает реальную способность канала генерировать выручку, а не просто интерес к продукту |
| **Алгоритм расчета** | COUNT(DISTINCT company_id с платежами) / COUNT(DISTINCT lead_id) * 100% |
| **Владелец** | Директор по маркетингу (по согласованию с Финансами) |

## Метрика 3: Время реакции поддержки (Support Response Time)

### Определения из разных источников

| Источник | Определение | Откуда взято |
|----------|-------------|---------------|
| Zendesk (Система) | Время до первого ответа оператора (First Reply) | 06_support_tickets.csv (поле response_time) |
| SLA (Регламент) | Время до полного закрытия тикета (Resolution Time) | 06_support_tickets.csv (resolved_date - created_date) |
| Клиентский опыт | Среднее время ожидания в очереди | Обсуждение команды |

### Количественный анализ

- Время до первого ответа: ~15 минут (быстро)
- Время до решения проблемы: ~28 часов (медленно)
- Расхождение: Значительное. Операторы отвечают быстро, но решение занимает время.

### Интерпретация

Для критерия "Низкая нагрузка на поддержку" (качество клиента) более показательно время *решения* проблемы. Клиенты с быстрым первым ответом, но долгим решением, создают повторные тикеты (Re-open), что увеличивает нагрузку. Метрика "Первый ответ" может вводить в заблуждение относительно реальной эффективности поддержки.

### Предложение единого определения

| Поле | Значение |
|------|----------|
| **Метрика** | Время реакции (Resolution Time) |
| **Определение** | Среднее время (в часах) от создания тикета до его закрытия (статус 'closed'), за вычетом времени ожидания ответа от клиента (статус 'pending') |
| **Обоснование** | Лучше отражает реальную нагрузку на поддержку и качество обслуживания. Быстрое закрытие тикета означает, что клиент доволен и не пишет повторно |
| **Алгоритм расчета** | AVG(resolved_date - created_date) WHERE status = 'closed' из файла 06_support_tickets |
| **Владелец** | Руководитель службы поддержки |

## Влияние на бизнес

Внедрение единых определений позволит:
1. Корректно оценить ROI рекламных кампаний, не завышая его за счет неоплаченных сделок.
2. Выявить каналы, которые приносят много лидов, но мало денег (низкая конверсия в оплату).
3. Понять, какие клиенты "дороги" в обслуживании (долгое время решения тикетов), и скорректировать стратегию привлечения.
