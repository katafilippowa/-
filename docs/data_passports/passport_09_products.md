# Паспорт данных: 09_products.csv

## 1. Общая информация
- **Имя файла:** 09_products.csv
- **Система-источник:** Google Sheets (ручное ведение)
- **Тип источника:** Справочник / НСИ
- **Владелец:** Продуктовый директор (CPO) / Менеджер продукта


## 2. Назначение и бизнес-контекст
Справочник продуктов и тарифов компании. Необходим для глубокого анализа эффективности каналов: некоторые каналы могут приводить клиентов, которые покупают только дешевые продукты, а другие — дорогие. Это влияет на расчет LTV и качества клиента.

## 3. Структура данных

| Поле | Тип | Описание | Пример | Обязательность | Примечания |
|------|-----|----------|--------|----------------|------------|
| product_id | int | ID продукта | 101 | Да | Primary Key |
| product_name | string | Название продукта | "Курс Python Pro" | Да | |
| product_category | string | Категория | "Программирование" | Да | |
| product_price | float | Стоимость | 15000.00 | Да | В рублях |
| is_active | bool | Актуальность | TRUE | Да | |
| description | string | Описание | TEXT | Нет | Не используется в аналитике |

## 4. Происхождение и обновление
- **Способ сбора:** Ручное заполнение в Google Sheets продуктовым менеджером
- **Частота обновления:** По мере запуска новых продуктов (в среднем раз в месяц)
- **Период хранения:** Текущий актуальный список + архив
- **Объем данных:** ~50 записей

## 5. Качество данных
- **Полнота:** 100% (справочник небольшой, контролируется вручную)
- **Уникальность ключей:** 100% (`product_id`)
- **Выявленные проблемы:**
  - Возможны человеческие ошибки при ручном вводе цен.
  - Отсутствие истории изменений цен (в файле только текущая цена).

## 6. Правила трансформации (ETL)
1. **Проверка целостности:** Проверка наличия `product_id` во всех транзакциях (если бы они были в этом файле, но у нас платежи в 05).
2. **Фильтрация:** Исключить неактивные продукты (`is_active = FALSE`) из текущего анализа, если не требуется исторический срез.
3. **Группировка:** Агрегация по `product_category` для анализа спроса по категориям в разрезе каналов.

## 7. Ограничения использования
- Данные могут быть неточными из-за ручного ввода.
- Цены являются розничными и могут отличаться от индивидуальных скидок (которые учитываются в платежах).
- Не содержит информации о себестоимости продукта.

## 8. Связи с другими источниками
- **Связан с:** `05_finance_payments.csv` (косвенно, если в платежах есть ссылка на продукт, но в текущей схеме Группа Б работает с агрегированными платежами).
- **Связан с:** `07_marketing_leads.csv` (если лиды атрибутируются по конкретному продукту в воронке).
- **Ключ связи:** `product_id`.
