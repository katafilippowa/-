# Паспорт данных: 07_marketing_leads.csv

## 1. Общая информация
- **Имя файла:** 07_marketing_leads.csv
- **Система-источник:** Маркетинговая платформа (Google Analytics, Facebook Ads, etc.)
- **Тип источника:** Операционная/маркетинг
- **Владелец:** Директор по маркетингу


## 2. Назначение и бизнес-контекст
Данные используются для отслеживания эффективности маркетинговых кампаний, анализа каналов привлечения лидов, расчета ROI и CAC. Критически важны для принятия решений о распределении маркетингового бюджета.

## 3. Структура данных

| Поле | Тип | Описание | Пример | Обязательность | Примечания |
|------|-----|----------|--------|----------------|------------|
| lead_id | int | Уникальный ID лида | 10001 | Да | Primary key |
| company_id | int | ID компании | 5001 | Да | Foreign key |
| lead_date | date | Дата создания лида | 2024-01-15 | Да | |
| channel | string | Канал привлечения | "Google Ads" | Да | Категориальное |
| campaign_name | string | Название кампании | "Summer Sale 2024" | Нет | |
| campaign_cost | float | Стоимость кампании | 50000.00 | Нет | В рублях |
| lead_source | string | Источник лида | "facebook.com" | Нет | UTM source |
| conversion_flag | bool | Флаг конверсии | TRUE/FALSE | Да | Вычисляемое |
| utm_source | string | UTM метка | "google" | Нет | |
| utm_medium | string | UTM метка | "cpc" | Нет | |
| utm_campaign | string | UTM метка | "brand_2024" | Нет | |

## 4. Происхождение и обновление
- **Способ сбора:** Автоматический импорт из рекламных платформ через API
- **Частота обновления:** Ежедневно (ночью)
- **Период хранения:** С 2019 года по настоящее время
- **Объем данных:** ~15,000 записей

## 5. Качество данных
- **Полнота:** 95% (пропуски в campaign_name и campaign_cost)
- **Уникальность ключей:** 100% (lead_id уникален)
- **Выявленные проблемы:**
  - Разные названия одного канала ("Google Ads", "google_adwords", "Google")
  - Пропуски в стоимости кампаний (15%)
  - Дубликаты лидов при кросс-канальной атрибуции

## 6. Правила трансформации (ETL)
1. **Нормализация каналов:** Привести все названия к единому справочнику
   - "Google Ads", "google_adwords", "Google" → "google_ads"
   - "Facebook", "fb", "facebook.com" → "facebook"
   
2. **Обработка пропусков:**
   - campaign_cost: заменить NULL на 0
   - campaign_name: заменить NULL на "Unknown_Campaign"
   
3. **Расчет conversion_flag:**
   - TRUE если company_id есть в 05_finance_payments с amount > 0
   
4. **Дедупликация:**
   - Оставить первую запись по lead_date при совпадении lead_id

## 7. Ограничения использования
- **Конфиденциальность:** Содержит информацию о маркетинговых бюджетах (коммерческая тайна)
- **Доступ:** Только для сотрудников аналитического и маркетингового отделов
- **Запрещено:** Передавать внешним подрядчикам без NDA
