# Паспорт данных: 05_finance_payments.csv

## 1. Общая информация
- **Имя файла:** 05_finance_payments.csv
- **Система-источник:** 1С:Бухгалтерия / 1С:Управление торговлей
- **Тип источника:** Финансовая/операционная
- **Владелец:** Финансовый департамент
- **Контактное лицо:** Финансовый директор / Руководитель отдела учета
- **Дата составления:** 2026-04-07
- **Составил:** Аналитик группы Б (удаленно)

## 2. Назначение и бизнес-контекст
Данные используются для учета всех финансовых транзакций с клиентами: поступления оплат, возвраты, отмены. Критически важны для:
- Расчета выручки и финансовых метрик (LTV, ARPU, MRR)
- Анализа платежного поведения клиентов
- Выявления просрочек и рисков оттока
- Формирования отчетности для руководства и инвесторов

## 3. Структура данных

| Поле | Тип | Описание | Пример | Обязательность | Примечания |
|------|-----|----------|--------|----------------|------------|
| payment_id | int | Уникальный ID платежа | 50001 | Да | Primary key, автоинкремент |
| company_id | int | ID компании-клиента | 1205 | Да | Foreign key → crm_companies |
| doc_date | date | Дата документа/платежа | 2024-03-15 | Да | Формат: YYYY-MM-DD |
| amount | float | Сумма платежа | 125000.00 | Да | В рублях, может быть отрицательной (возврат) |
| payment_type | string | Тип оплаты | "card", "transfer", "invoice" | Да | Справочник из 1С |
| status | string | Статус платежа | "paid", "refunded", "cancelled" | Да | Фильтровать только "paid" для выручки |
| currency | string | Валюта | "RUB" | Да | Пока только рубли |
| invoice_id | int | ID счета | 8821 | Нет | Может быть NULL для мелких платежей |
| payment_method | string | Способ оплаты | "online", "bank", "cash" | Нет | Заполняется не всегда |
| created_at | datetime | Дата создания записи | 2024-03-15 14:30:00 | Да | Техническое поле |

## 4. Происхождение и обновление
- **Способ сбора:** Автоматический экспорт из 1С через nightly job
- **Частота обновления:** Ежедневно, в 03:00 MSK
- **Период хранения:** С 2019-01-01 по настоящее время
- **Объем данных:** ~35,000 записей (оценка)
- **Размер файла:** ~4.2 MB (CSV)

## 5. Качество данных
- **Полнота:** 98.5% (пропуски в `invoice_id` и `payment_method`)
- **Уникальность ключей:** 100% (`payment_id` уникален)
- **Целостность:** 99.2% (`company_id` существует в CRM для 99.2% записей)
- **Выявленные проблемы:**
  - Отрицательные суммы в `amount` (возвраты) — требуют отдельной обработки
  - Дубликаты при повторном импорте (решается по `payment_id + created_at`)
  - Разные форматы дат в исторических данных (исправлено в ETL)
