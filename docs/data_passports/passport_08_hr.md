# Паспорт данных: 08_hr_employees.csv

## 1. Общая информация
- **Имя файла:** 08_hr_employees.csv
- **Система-источник:** HR-система (1С:ЗУП / BambooHR)
- **Тип источника:** Справочник / Нормативно-справочная информация (НСИ)
- **Владелец:** HR-директор (CHRO)


## 2. Назначение и бизнес-контекст
Справочник сотрудников компании. Используется для определения владельцев данных (например, кто отвечает за каналы маркетинга или обработку тикетов) и для обогащения данных о поддержке (какой отдел генерирует больше всего тикетов).

## 3. Структура данных

| Поле | Тип | Описание | Пример | Обязательность | Примечания |
|------|-----|----------|--------|----------------|------------|
| employee_id | int | Табельный номер | 203 | Да | Primary Key |
| full_name | string | ФИО сотрудника | Иванов И.И. | Да | ПДн |
| department | string | Департамент | "Маркетинг" | Да | |
| position | string | Должность | "Менеджер" | Да | |
| hire_date | date | Дата приема | 2022-03-01 | Да | |
| termination_date | date | Дата увольнения | NULL | Нет | NULL = активен |
| is_active | bool | Статус занятости | TRUE | Да | Расчетное |
| manager_id | int | ID руководителя | 101 | Нет | |

## 4. Происхождение и обновление
- **Способ сбора:** Выгрузка из HR-системы
- **Частота обновления:** Еженедельно (при кадровых изменениях)
- **Период хранения:** История с 2019 года
- **Объем данных:** ~350 записей

## 5. Качество данных
- **Полнота:** 98% (иногда отсутствует `manager_id` для топ-менеджмента)
- **Уникальность ключей:** 100% (`employee_id`)
- **Выявленные проблемы:**
  - Разные написания департаментов ("Отдел продаж", "Sales", "Продажи").
  - Устаревшие записи уволенных сотрудников, если не фильтровать по `is_active`.

## 6. Правила трансформации (ETL)
1. **Фильтрация активных:** Отбирать только `is_active = TRUE` или `termination_date IS NULL` для текущих отчетов.
2. **Нормализация департаментов:** Приведение к единому справочнику: `{'sales': 'Продажи', 'marketing': 'Маркетинг'}`.
3. **Маппинг владельцев:** Создание таблицы соответствия `department` → `owner_name` для глоссария.

## 7. Ограничения использования
- Строго конфиденциальные данные (ПДн).
- Запрещено использование ФИО в общих аналитических витринах (использовать только ID или обезличенные роли).
- Доступ только для HR и руководства.
