# Журнал допущений и проблем. Группа Б
## Прогнозирование эффективности маркетинговых каналов
**Группа:** Б (удаленная работа)  
**Задача:** Прогноз лидов, конверсий и выручки по каналам на апрель 2025

---

## Допущения

### Допущение 1: Связывание лидов с компаниями по названию
- **Суть:** Для связи таблицы лидов (07_marketing_leads.csv) с таблицей компаний (01_crm_companies.csv) используется поле `company_name` без дополнительной нормализации.
- **Обоснование:** В исходных данных отсутствует прямой ключ `company_id` в таблице лидов. Нормализация названий (удаление ООО/АО, исправление опечаток) требует дополнительных ресурсов и может внести субъективные ошибки.
- **Потенциальный риск:** 
  - Лиды от одной компании могут быть связаны с разными записями в CRM из-за вариаций в написании названия
  - Новые компании, еще не добавленные в CRM, останутся несвязанными
- **Влияние на результат:** 
  - Занижение LTV по каналам (несвязанные лиды не учитываются в расчете выручки)
  - Возможное завышение CAC (затраты делятся на меньшее число конверсий)
- **Мера снижения риска:** В итоговом отчете явно указывать долю несвязанных лидов (~15-20% в тестовых данных) и интерпретировать их как "потенциальные новые клиенты".

---

### Допущение 2: Определение канала привлечения по первому лиду
- **Суть:** Канал привлечения компании определяется по самому раннему лиду (минимальная `lead_date`). Все последующие взаимодействия с этой компанией атрибутируются тому же каналу.
- **Обоснование:** В B2B-сегменте цикл принятия решения длительный, и первый контакт чаще всего определяет источник привлечения. Атрибуция по последнему клику (last-click) завысит роль ретаргетинга и контекстной рекламы.
- **Потенциальный риск:** Если клиент впервые узнал о продукте через один канал, но конвертировался после взаимодействия с другим, атрибуция будет некорректной.
- **Влияние на результат:** 
  - Возможное занижение ROI каналов верхнего уровня воронки (organic, referral)
  - Возможное завышение ROI каналов нижнего уровня (retargeting, email)
- **Мера снижения риска:** В будущих итерациях рассмотреть модель multi-touch атрибуции, если данные позволят.

---

### Допущение 3: Стационарность конверсии во времени
- **Суть:** Конверсия из лида в продажу (`conversion_rate`), рассчитанная по историческим данным, считается постоянной для прогноза на апрель 2025.
- **Обоснование:** В условиях ограниченных данных и отсутствия явных сезонных паттернов в краткосрочной перспективе (1 месяц) допущение о стабильности конверсии является разумным компромиссом между сложностью и точностью.
- **Потенциальный риск:** Если в апреле запланированы маркетинговые активности, изменяющие качество лидов (например, акция для "холодной" аудитории), фактическая конверсия может отклониться от прогноза.
- **Влияние на результат:** 
  - При ухудшении качества лидов: прогноз продаж будет завышен
  - При улучшении качества лидов: прогноз продаж будет занижен
- **Мера снижения риска:** В меморандуме явно указать, что прогноз конверсии чувствителен к изменениям в стратегии привлечения.

---

### Допущение 4: Независимость наблюдений для биномиальной модели
- **Суть:** Конверсия каждого лида считается независимым событием с фиксированной вероятностью `p = conversion_rate`.
- **Обоснование:** Биномиальное распределение — стандартный и интерпретируемый способ моделирования неопределенности в воронке продаж. Учет корреляций между лидами (например, от одной компании) требует более сложных моделей и данных, которых нет в распоряжении.
- **Потенциальный риск:** Если несколько лидов от одной компании конвертируются/не конвертируются совместно, дисперсия числа продаж будет недооценена.
- **Влияние на результат:** Доверительные интервалы для числа продаж могут быть уже реальных.
- **Мера снижения риска:** Использовать консервативную оценку стандартной ошибки (умножение на коэффициент 1.2-1.5) при расчете интервалов.

---

### Допущение 5: Адекватность модели Хольта для прогноза лидов
- **Суть:** Экспоненциальное сглаживание с трендом (метод Хольта) адекватно описывает динамику числа лидов по каждому каналу.
- **Обоснование:** Метод Хольта прост в реализации, интерпретируем и хорошо работает при наличии тренда без выраженной сезонности. Для краткосрочного прогноза (1 месяц) этого достаточно.
- **Потенциальный риск:** Если в данных есть скрытая сезонность или структурные сдвиги, прогноз может быть смещенным.
- **Влияние на результат:** Прогноз числа лидов может быть завышен или занижен, что повлияет на все последующие расчеты.
- **Мера снижения риска:** Использовать консервативную оценку ошибки прогноза (30-40% от значения при малом объеме данных) и явно маркировать каналы с "низкой достоверностью".

## Выявленные проблемы

### Проблема 1: Пропуски в поле `cost` для лидов
- **Описание:** В файле 07_marketing_leads.csv поле `cost` имеет пропуски (~15% записей), особенно для органических каналов.
- **Решение:** Заменить пропуски на 0 для каналов, где стоимость по определению отсутствует (organic, referral). Для платных каналов с пропусками — использовать медианное значение по каналу за период.
- **Влияние:** При некорректной обработке может быть занижен CAC и завышен ROI.

---

### Проблема 2: Разные форматы дат в источниках
- **Описание:** Даты в разных файлах представлены в разных форматах (DD.MM.YYYY, YYYY-MM-DD, datetime с часами).
- **Решение:** Привести все даты к единому формату `datetime64[ns]` с явным указанием часового пояса (МСК, UTC+3).
- **Влияние:** Ошибки в расчете временных окон (90 дней для конверсии, 24 месяца для LTV).

---

### Проблема 3: Открытые тикеты в данных поддержки
- **Описание:** В файле 06_support_tickets.csv поле `resolved_date` имеет пропуски для открытых тикетов (~10% записей).
- **Решение:** При расчете времени реакции учитывать только тикеты со статусом `closed` или `solved`. Для оценки текущей нагрузки использовать отдельную метрику "открытые тикеты".
- **Влияние:** Без фильтрации среднее время решения будет завышено.

---

### Проблема 4: Дубликаты лидов при кросс-канальной атрибуции
- **Описание:** Один и тот же пользователь может оставить несколько лидов через разные каналы, что приводит к дублированию `company_name`.
- **Решение:** При связывании с компаниями оставлять запись с минимальной `lead_date` (первый контакт) как определяющую канал привлечения.
- **Влияние:** Без дедупликации может быть завышено число лидов и занижена конверсия.

## Ловушки анализа

### Ловушка 1: Скошенное распределение LTV
- **Суть:** Выручка от клиентов (LTV) имеет сильную правостороннюю асимметрию: большинство клиентов приносят умеренный доход, но есть единичные крупные контракты, "тянущие" среднее вверх.
- **Проблема:** Использование нормального приближения для построения доверительных интервалов среднего LTV приводит к:
  - Включению отрицательных значений в нижнюю границу (экономически бессмысленно)
  - Симметричным интервалам, не отражающим реальную асимметрию распределения
- **Как избежали:** 
  - Применили непараметрический метод бутстрапа для оценки распределения среднего LTV
  - Использовали логнормальное распределение для симуляции индивидуальных значений LTV (при отсутствии реальных данных)
  - Доверительные интервалы рассчитывались как 2.5-й и 97.5-й процентили бутстрап-распределения
- **Вывод:** Получены асимметричные, экономически осмысленные интервалы для LTV, которые корректно отражают риск "тяжелого хвоста".

---

### Ловушка 2: Двухуровневая неопределенность в прогнозе выручки
- **Суть:** При прогнозе выручки есть два источника случайности:
  1. Неопределенность в числе лидов (прогноз методом Хольта имеет ошибку)
  2. Случайность конверсии (даже при фиксированном числе лидов не все конвертируются)
- **Проблема:** Если учитывать только одну из них (например, только биномиальную неопределенность конверсии), доверительные интервалы выручки будут существенно уже реальных, что создаст ложное чувство уверенности в прогнозе.
- **Как избежали:** 
  - Использовали формулу полной дисперсии: `Var(sales) = Var(n)*p² + E[n]*p*(1-p)`
  - Для LTV дополнительно применили бутстрап, чтобы учесть асимметрию распределения
  - При расчете интервала выручки использовали консервативное перемножение границ: `(sales_lower * ltv_lower, sales_upper * ltv_upper)`
- **Вывод:** Доверительные интервалы выручки отражают совокупную неопределенность на всех этапах воронки, что позволяет принимать более обоснованные решения о распределении бюджета.

---

### Ловушка 3: Экстраполяция тренда при малом объеме данных
- **Суть:** Для некоторых каналов в исторических данных (12_marketing_history.csv) может быть менее 4-5 точек, что недостаточно для надежной оценки параметров модели Хольта.
- **Проблема:** Прямая экстраполяция в таких случаях приводит к неустойчивым прогнозам с заниженной ошибкой.
- **Как избежали:** 
  - Ввели порог: если точек < 4, используем простое среднее за последние 3 месяца как прогноз
  - Стандартную ошибку в этом случае оцениваем как 30-40% от прогноза (консервативная оценка)
  - В отчете явно помечаем каналы с "низкой достоверностью прогноза"
- **Вывод:** Прогнозы для каналов с малым объемом данных имеют широкие доверительные интервалы, что корректно отражает высокую неопределенность.

---

### Ловушка 4: Игнорирование корреляции между метриками
- **Суть:** Конверсия и LTV могут быть коррелированы: каналы, привлекающие более качественных лидов, могут иметь и более высокую конверсию, и более высокий LTV.
- **Проблема:** При независимом моделировании конверсии и LTV доверительные интервалы выручки могут быть некорректными.
- **Как избежали:** 
  - В текущей работе использовали консервативное перемножение границ (наихудший/наилучший сценарий)
  - В будущих итерациях рассмотреть совместное моделирование через многомерный бутстрап
- **Вывод:** Текущий подход дает "запас прочности", но может быть избыточно консервативным.

## Рекомендации для следующих итераций

1. **Улучшить связывание лидов и компаний:** 
   - Внедрить fuzzy matching для названий компаний
   - Добавить поле `company_id` в таблицу лидов на этапе сбора данных

2. **Сбор данных для multi-touch атрибуции:** 
   - Фиксировать все точки контакта клиента с маркетингом
   - Внедрить UTM-метки на всех этапах воронки

3. **Мониторинг конверсии в реальном времени:** 
   - Настроить дашборд с еженедельным обновлением `conversion_rate` по каналам
   - Внедрить алерты при отклонении конверсии >2σ от исторического среднего

4. **Расширение исторических данных:** 
   - Запросить у маркетинга данные за более длительный период (минимум 12 месяцев)
   - Это повысит надежность прогнозов методом Хольта для всех каналов

5. **Совместное моделирование метрик:** 
   - Рассмотреть многомерный бутстрап для учета корреляции между конверсией и LTV
   - Использовать байесовские методы для более гибкого учета неопределенности
